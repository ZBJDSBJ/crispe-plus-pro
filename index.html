01 · Framework

六维框架解析

CRISPE+ 通过六个维度构建完整的 AI 指令体系，每张卡片包含详细说明与真实例句。

C
Capacity & Role
能力与角色
赋予 AI 一个具体的专家身份。角色越精确，AI 的思维框架越专业。可以叠加多重角色与背景。
"你是一位有 10 年经验的 UX 设计师，同时具备用户心理学背景，曾主导多个 SaaS 产品的体验改版。"
R
Insight & Context
背景与见解
提供任务的背景信息：你是谁、项目状态、受众特征、已知限制。背景越充分，回答越贴合。
"我负责一款面向企业客户的 B2B SaaS 产品，目前用户留存率偏低，主要用户是 IT 部门负责人。"
I
Statement & Task
任务陈述
核心指令。明确告诉 AI 要做什么。用动词开头，可以拆分成步骤，越具体越好。
"请为我的产品设计一套 3 步骤的新用户引导流程，重点解决用户注册后第一周的流失问题。"
S
Personality & Style
个性与风格
指定回答的语气、风格、措辞习惯。可以参考具体人物、品牌，或描述目标读者的阅读感受。
"语气专业但不失亲切，像顾问而非教授。避免行话，多用类比，让非技术背景的高管也能理解。"
P
Experiment
实验与示例
要求 AI 提供多个版本、角度或对比选项。让你在不同方案中快速筛选最优解。
"请给出 3 个版本：一个保守方案、一个激进创新方案、一个折中方案，标注核心差异。"
E+
Format (Enhanced)
输出格式（增强）
指定输出形式：Markdown、表格、JSON、代码、流程图描述、编号列表……格式明确让输出直接可用。
"用 Markdown 输出，每个方案用 ## 标题，包含「核心思路」「实施步骤」「预期风险」三个子章节。"

07 · Cheatsheet

速查手册

💡 角色设定技巧
叠加职业 + 专长 + 经验年限
加入行业背景：「服务过500强」
指定知识边界：「专注B端产品」
用「同时具备」叠加跨学科角色
避免过宽：「AI专家」< 「CV算法工程师」
📝 任务描述技巧
用动词开头：写/分析/设计/评估
加数量约束：3条/500字/5个方案
指定交付物：「表格/流程图/代码」
分步骤拆解复杂任务
加「不要」约束排除不需要的内容
🎯 提升效果技巧
加「先思考，再输出」提升推理
用「假设你的用户是…」锁定受众
「列出你的假设」让逻辑透明
「给出置信度」量化AI的确定性
「逐步验证」让AI自我检查输出
⚡ 常用格式模板
用Markdown，## 分节，**加粗**重点
表格对比：| 方案 | 优点 | 缺点 |
JSON结构化数据输出
「先给结论，再展开」倒金字塔
「每条不超过X字」控制输出长度
🚫 常见错误
只说「帮我写篇文章」太模糊
角色太宽泛：「你是一个助手」
忘记说明目标受众是谁
没有限制输出长度和格式
任务和背景混在一起写
🔄 迭代优化流程
第1轮：先用最简版测试方向
第2轮：补充缺失的背景信息
第3轮：调整风格和输出格式
第4轮：加入约束条件和反例
收藏最终版本建立个人提示词库

06 · Project Doc Files

五种项目规范文件

Cursor、Windsurf、Claude Code 等 AI 编辑器通过项目规范文件理解你的代码库和工作方式。Rules / Skills / Plans / Specs / Agent 各有不同职责，组合使用可让 AI 在整个项目中保持一致的行为。

✦ 五种文件如何叠加
每种文件各司其职，单独使用都有效。叠加使用时，建议按以下顺序思考，从「边界」到「标准」逐层收紧对 AI 的定义。
① Rules
先划边界
不能做什么
② Skills
再赋能力
擅长什么
③ Plans
然后定流程
怎么执行
④ Specs
再定标准
输出要求
⑤ Agent
最后封装
可复用角色
